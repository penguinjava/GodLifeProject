## 📌 Git Commit 메시지 규칙

모든 커밋 메시지는 아래 규칙을 따라 작성합니다.


| 타입       | 설명                                           |
|------------|------------------------------------------------|
| `feat`     | 새로운 기능 추가                               |
| `fix`      | 버그 수정                                      |
| `docs`     | 문서 관련 수정 (README, 주석 등)               |
| `style`    | 코드 포맷팅, 세미콜론 누락, 들여쓰기 등 (비기능적) |
| `refactor` | 코드 리팩토링 (기능 변화 없이 구조 개선)       |
| `chore`    | 기타 잡일, 빌드 설정 변경 등                   |
